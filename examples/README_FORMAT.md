## Label Format (YOLO format)
Each `.txt` file contains bounding boxes: one per line
```
<class_id> <x_center> <y_center> <width> <height>
```
All normalized to [0, 1].

Example `image_001.txt`:
```
0 0.5 0.3 0.2 0.1
1 0.7 0.6 0.15 0.2
```

## data.yaml format (for Ultralytics)
```yaml
path: /absolute/path/to/data
train: images/train
val: images/val
test: images/test

nc: 3  # number of classes
names: ['class0', 'class1', 'class2']
```

## Results CSV Format
After running `optimize_and_validate_grid.py`, each corruption family produces a CSV:
- `outputs/grid_eval_<timestamp>/<family>.csv`

Columns:
- `ts`: Unix timestamp
- `family`: Corruption family name
- `params_json`: JSON string of parameters
- `hue`: Whether hue jitter was applied (0 or 1)
- `seed`: Random seed
- `model`: Model name
- `gflops`: Floating-point operations (billion)
- `params`: Number of model parameters
- `gpu_mem_mb`: Peak GPU memory (MB)
- `P`, `R`: Precision and Recall
- `mAP50`, `mAP5095`: Mean Average Precision metrics
- `inf_ms`: Inference time (milliseconds)
- `FPS`: Frames per second
